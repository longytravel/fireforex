## What's BROKEN / INCOMPLETE (user-flagged)

1. **Backtest ↔ live parity NOT tight yet.** Sources of drift:
   - Feed mismatch: backtest runs off Dukascopy M1, live uses IC Markets M1.
   - Execution timing: engine assumes fill at M1 open, live fills ~30s–2s after bar close (MT5 only publishes M1 ~30s after the minute closes; runner polls 1s).
   - Spread model: engine reads Dukascopy `spread` column, live uses real bid/ask. IC Markets spread often much tighter on majors.
   - **Engine fix required:** add IC Markets M1 parquet ingest, plus an "execution_delay_bars" knob on the backtest so it fills one M1 bar later than the signal bar close. Wire into `simulate_trade_full`.

2. **Reconciler produces empty reports because backtest trade log has no `pair` column.** Pair is only on the live plans. Need to emit `pair` on every backtest trade row. Harness currently saves `trades` structured array from `ff/harness._build_best_trade_log()` — add `pair` field populated from `ea["data"]["pair"]`.

3. **Multi-pair parity.** Current Deploy writes one best-trial set of params for all pairs. Each pair actually wants its own calibration (different ATR regimes). Phase E.1 has `scripts/calibrate_for_parity.py` — not yet wired to multi-pair service_config. Path forward: calibrate each pair separately, merge per-pair override dicts on top of the shared recipe.

4. **Live "speed" concern.** MT5 M1 bars are only returned ~30s+ after the minute closes; poll interval is 1s. Can't go lower without subscribing to ticks (`mt5.symbol_info_tick` loop). Not planned for v1 — reconciler is timestamp-matched, not wall-clock-matched, so this is a perception issue more than a parity issue, *provided the backtest also fills at the same lagged timestamp* (see point 1).

5. **Laptop browser needs Ctrl+F5 after every code pull.** The Restart bat kills the server and opens a NEW tab, but any pre-existing tab still shows stale cached JS. Minor, but user doesn't know to hit Ctrl+F5.

6. **Pre-existing crashes.jsonl shows the first-ever startup crash** (fixed in commit 2f03e67). Not a current bug. User should be told it's historical.

## Known quirks to forewarn the user

- IC Markets demo account MT5 broker time runs GMT+2/+3 (DST-aware). Fixed on connect.
- MT5 symbols match Fire Forex pair names straight (`EUR_USD` → `EURUSD`). If broker ever uses a suffix variant (`.a`, `.i`, `-cent`), plans would fire but orders fail — would surface in `artifacts/live/errors.jsonl` as "no tick for X".
- Scheduled Task `ff-live-runner` restart on failure every 60s. So if the process dies, it self-heals within a minute. Visible in `schtasks /Query /TN ff-live-runner` under `Status`.
- Auto-reconcile thread fires every 60 min by default (`auto_reconcile_interval_min` in service_config). Writes `artifacts/live/reconcile/<stamp>.html` + `latest.html`.
