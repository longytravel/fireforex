## What's broken

**Runner init deadlock.** Symptom:
- I kicked `ff-live-runner` twice (PIDs 9368 then 10688).
- Both times the process went to **CPU=0, WorkingSet=3.2 MB** and stayed
  that way for 50-60+ minutes.
- No `state.json` appeared (runner writes it per poll iteration — so
  poll loop never starts).
- No `errors.jsonl`, no `crashes.jsonl`, no traceback. Silent hang.
- WS=3.2 MB is **far too small** for a Python 3.12 process with imports;
  it looks like a kernel-wait state, not slow warmup.

Likely causes (ordered):
1. **`runner_service.py` main() blocks inside `runner.run(cfg, stop_event)`**
   before first poll, on something that doesn't log before blocking
   (e.g. `mt5.initialize`, Dukascopy download for 28 pairs, a data-prime
   loop with no progress logging).
2. **MT5 terminal mutex** — if another Python process holds the MT5
   session, runner's `initialize` can block. (My diag query took the
   lock briefly each time it ran — see "MT5 responsive" below.)
3. **`C:\ff-scripts\ff-live.bat`** — the actual thing the scheduled
   task runs — has **no stdout capture**, so any early print/log is
   invisible.

MT5 terminal is responsive: my diag's `mt5.initialize` returns and
`positions_get` works. So the terminal process is up.

### Fix A — patch `ff/live/runner_service.py` to log to file

Small edit in `main()`: add a `logging.FileHandler` writing to
`artifacts/live/runner.log` **before** any import-from-ff runs.
Emit first line `"[svc] starting runner_service"` straight away so
**any** hang leaves a breadcrumb. This is the fix I'd ship.

Path: `C:\Users\ROG\Projects\Fire Forex\ff\live\runner_service.py`,
function `main()` around line 57. Currently:
```python
def main() -> int:
    logging.basicConfig(
        level=logging.INFO,
        format="%(asctime)s [%(levelname)s] %(name)s: %(message)s",
    )
```
Change to:
```python
def main() -> int:
    _LIVE_DIR.mkdir(parents=True, exist_ok=True)
    _log_file = _LIVE_DIR / "runner.log"
    logging.basicConfig(
        level=logging.INFO,
        format="%(asctime)s [%(levelname)s] %(name)s: %(message)s",
        handlers=[
            logging.FileHandler(_log_file, mode="w", encoding="utf-8"),
            logging.StreamHandler(),
        ],
    )
    LOG.info("[svc] starting runner_service; log=%s", _log_file)
```
